# 勤怠管理システム (Work Management System)

## 📌 プロジェクト概要

社内社員の出退勤記録およびお知らせを管理するためのWebバックエンドシステムです。
JWT認証を導入しセキュリティを確保するとともに、ドメイン単位の構造で設計し保守性を向上させました。

---

## 🧩 主な機能

* ログイン / 認証（JWT Authentication）
* 出勤・退勤記録管理
* ユーザー情報取得
* お知らせ閲覧

---

## 🏗️ アーキテクチャ設計

本プロジェクトは機能単位ではなく、**ドメイン中心設計（Domain-Based Structure）** を採用しています。

各ドメインは以下の層で分離しています。

* Controller : リクエスト/レスポンス処理
* Service : ビジネスロジック
* Mapper : DBアクセス（MyBatis）
* DTO : 画面転送データ
* Entity : ドメインデータ

これによりビジネスロジックと参照処理を分離し、
拡張性と保守性を考慮した構造にしました。

---

## 🔐 認証方式

セッション方式ではなくJWTトークン認証を採用しました。

* ステートレス構造の維持
* モバイル・外部API拡張を考慮
* フィルターベースの認証処理

---

## 🛠️ 技術スタック

* Java 17
* Spring Boot
* Spring Security
* JWT
* MyBatis
* Oracle Database
* Gradle

---

## 💡 設計のポイント

* DTOとEntityの分離によるドメイン安定性確保
* 認証ロジックをFilterへ分離し責務を明確化
* 機能単位パッケージ構造による保守性向上

---

## 📈 学んだこと

単純な機能実装よりも設計が保守性に大きく影響することを学びました。
特に認証処理とドメイン分離を通じて、拡張可能なサーバー構造について考える経験になりました。
